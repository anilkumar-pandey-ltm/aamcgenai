# Multi-Language Service Client Generator Guide

## 🤖 Agent Information

**Agent Mode**: `api-service-client-generator`  
**Agent File**: `.github/agents/api-service-client-generator.agent.md`  
**Activation**: Use `@api-service-client-generator` prefix in your Copilot prompts

### How to Activate This Agent

```
@api-service-client-generator [Your Prompt]
```

**Example:**
```
@api-service-client-generator Generate API service client for user_management API configuration: "tests/apidata/user_management/"
```

## 📘 Overview

This guide explains how to use the **Multi-Language Service Client Generator Agent** to automatically generate framework-aware, production-ready API service clients for any programming language.

---

## 🎯 What This Agent Does

The Multi-Language Service Client Generator Agent:
- ✅ Automatically detects your framework's programming language (TypeScript, Python, Java, C#, Go, Kotlin)
- ✅ Generates API service clients that extend existing base classes
- ✅ Integrates with your framework's logging and configuration utilities
- ✅ Incorporates business rules and domain models from MCP context servers
- ✅ Supports multiple authentication methods (Bearer, Basic, API Key, OAuth2, JWT)
- ✅ Creates a single, focused production-ready client file
- ✅ Saves to configured output path automatically

---

## 🔧 Prerequisites

### Required Setup

1. **MCP Servers Running**:
   - `mcp_context_server.py` - For business rules and domain context
   - `mcp_automation_server.py` - For framework patterns and templates

2. **API Configuration Files**:
   You need YAML files generated by the API Analyzer Agent:
   - `endpoints.yaml` - API endpoint definitions
   - `schema.yaml` - Request/response schemas
   - `testdata.yaml` - Test data samples
   - `template.yaml` - Dynamic data templates

3. **Output Path Configured**:
   - Path defined in `copilot-agent.paths.yaml` as `{{test_paths.serviceclient}}`
   - Default: `tests/serviceclient/`

## 💡 How It Works

### What Copilot Does Automatically

When you provide a prompt, the agent:

**Step 1: Framework Assessment**
```
✓ Scans project structure
✓ Detects primary programming language (TypeScript/Python/Java/C#)
✓ Identifies existing base client classes (ApiClient, BaseAPIClient)
✓ Analyzes framework patterns and conventions
✓ Locates logging and configuration utilities
```

**Step 2: MCP Context Retrieval**
```
✓ Connects to mcp_context_server
✓ Retrieves business rules from {{data_paths.context_business_rules}}
✓ Retrieves domain models from {{data_paths.context_domain}}
✓ Retrieves application context from {{data_paths.context_application}}
✓ Identifies validation requirements
```

**Step 3: API Configuration Analysis**
```
✓ Reads endpoints.yaml - Maps all API endpoints
✓ Reads schema.yaml - Understands request/response structures
✓ Reads testdata.yaml - Identifies data patterns
✓ Reads template.yaml - Plans dynamic data handling
✓ Extracts authentication requirements
```

**Step 4: Code Generation**
```
✓ Generates language-specific service client
✓ Extends existing base client class
✓ Implements all endpoint methods
✓ Adds TypeScript interfaces/types
✓ Integrates business rule validations
✓ Implements authentication mechanisms
✓ Adds error handling patterns
✓ Includes logging integration
```

**Step 5: File Creation**
```
✓ Creates single service client file
✓ Saves to {{test_paths.serviceclient}}/
✓ Validates syntax and structure
✓ Confirms successful generation
```

## 🎓 Best Practices

### DO's ✅

1. **Always Prepare API Configuration First**: Use API Analyzer Agent to generate YAML files
2. **Specify Service Name Clearly**: Use descriptive names like "PrestashopProducts", "CustomerManagement"
3. **Let Agent Auto-Detect Language**: Don't specify language unless you need to override detection
4. **Review Generated Code**: Always validate the generated client for your specific needs
5. **Use Existing Base Classes**: Generated clients extend framework base classes for consistency
6. **Check MCP Servers**: Ensure both context servers are running for best results

### DON'Ts ❌

1. **Don't Create Manual Clients**: Let the agent generate to ensure consistency
2. **Don't Hardcode URLs**: Agent uses framework configuration (base-urls.yaml)
3. **Don't Skip Business Rules**: Allow agent to incorporate validation logic
4. **Don't Modify Base Classes**: Generated clients should only extend, not modify base classes
5. **Don't Generate Without YAML Files**: Always have endpoints.yaml and schema.yaml ready

## 🔧 Troubleshooting

### Issue 1: MCP Servers Not Running
**Error**: "MCP Context Server is not running"

**Solution**:
```powershell
# Start MCP Context Server
python src/mcp/mcp_context_server.py

# Start MCP Automation Server
python src/mcp/mcp_automation_server.py
```

### Issue 2: API Configuration Not Found
**Error**: "Cannot find endpoints.yaml in specified directory"

**Solution**: Run API Analyzer first
```
Analyze API document from [YOUR_INPUT_PATH]\[your_api_document].json and save it in [YOUR_OUTPUT_PATH]
```

### Issue 3: Framework Language Not Detected
**Error**: "Framework language not detected"

**Solution**: Manually specify language
```
Generate [Python/Java/C#] API service client from [YOUR_API_CONFIG_PATH]/

Target Language: [Python/Java/C#/TypeScript]
```

### Issue 4: Base Client Class Not Found
**Warning**: "Base client class not found, generating standalone implementation"

**Solution**: 
- Check if `framework/api/clients/apiClient.ts` exists
- Or let agent generate standalone client
- Or specify base class manually in prompt
